🏗️ L'Architecture du Web — Ce que personne n'explique

Avant d'écrire une seule ligne de JavaScript, tu dois comprendre comment le navigateur fonctionne. C'est comme apprendre à conduire : tu dois savoir ce qu'il y a sous le capot.

┌─────────────────────────────────────────────────────┐ │ TON NAVIGATEUR (Chrome, Firefox) │ │ │ │ ┌─────────────┐ ┌──────────────┐ ┌───────────┐ │ │ │ HTML │ │ CSS │ │ JS │ │ │ │ Le squelette│ │ Le maquillage│ │ Le cerveau│ │ │ │ de la page │ │ les couleurs │ │ l'action │ │ │ └──────┬──────┘ └──────┬───────┘ └─────┬─────┘ │ │ │ │ │ │ │ ▼ ▼ ▼ │ │ ┌─────────────────────────────────────────────┐ │ │ │ DOM (Document Object Model) │ │ │ │ Représentation mémoire de toute la page │ │ │ │ C'est l'arbre que JavaScript manipule │ │ │ └─────────────────────────────────────────────┘ │ │ │ │ │ ▼ │ │ ┌─────────────────────────────────────────────┐ │ │ │ Moteur JavaScript (V8) │ │ │ │ Lit et exécute ton code ligne par ligne │ │ │ └─────────────────────────────────────────────┘ │ └─────────────────────────────────────────────────────┘

DOM = Document Object Model. Inventé par le W3C en 1998. Document = la page web. Object = représenté comme des objets en mémoire. Model = c'est un modèle (une représentation) — pas la vraie page, mais sa copie en mémoire que JS peut manipuler.

Imagine une maison : HTML = les murs et la structure, CSS = la peinture et la déco, JavaScript = l'électricité et la domotique (lumières qui s'allument, portes qui s'ouvrent). Le DOM = le plan architectural que l'électricien consulte pour savoir où brancher quoi.

LEÇON 02 variables · types · opérateurs
Variables & Types de Données
Comprends comment JavaScript stocke et gère l'information en mémoire.
📦 C'est quoi une variable ?

Une variable, c'est une boîte avec un nom dans la mémoire de l'ordinateur. Tu mets quelque chose dedans, et tu peux le retrouver plus tard grâce à son nom.

C'est comme les contacts dans ton téléphone. "Maman" est un nom (la variable) qui pointe vers un numéro (la valeur). Tu n'as pas besoin de mémoriser le numéro — tu utilises juste le nom.

var = abréviation de variable (latin : variabilis = qui peut changer). let = mot anglais signifiant "laisser/permettre" → "laisse-moi appeler ça X". const = abréviation de constant (latin : constare = rester fixe) → une valeur qui ne change jamais.

variables.js
// ── LET : Variable qui peut changer ── let prenom = "Moussa"; let age = 20; prenom = "Fatou"; // ✅ on peut changer age = 21; // ✅ ok // ── CONST : Variable qui ne change PAS ── const pays = "Sénégal"; // pays = "Mali"; ❌ ERREUR ! const ne change pas // ── VAR : L'ancien système (évite-le) ── var ancienStyle = "déprécié"; // var a des bugs de portée — utilise let/const console.log(prenom); // "Fatou" console.log(age); // 21 console.log(pays); // "Sénégal"
🎯 Les Types de Données

JavaScript a 7 types primitifs. Comprends-les bien, ils sont la base de tout.

TypeValeur exempleUsage
String "Dakar", 'salam' Texte — toujours entre guillemets
Number 42, 3.14, -100 Nombres entiers et décimaux
Boolean true, false Oui/Non, Vrai/Faux
null null Vide intentionnel ("la boîte est vide exprès")
undefined undefined Pas encore défini ("la boîte n'a jamais été remplie")
BigInt 9007199254740991n Très grands nombres
Symbol Symbol('id') Identifiant unique (avancé)

String = "chaîne" en anglais — une chaîne de caractères. Boolean = du nom de George Boole (1815-1864), mathématicien qui inventa la logique binaire (vrai/faux). null = latin nullus = "rien du tout". undefined = anglais "non défini" — JS n'a pas encore de valeur pour cette variable.

LEÇON 03 functions · scope · arrow
Les Fonctions — Le Cœur de JS
Une fonction = un bloc de code réutilisable avec un nom. C'est la clé de tout programme.
🔧 Définir & appeler une fonction

function = du latin functio = "accomplir une tâche". En maths, f(x) = 2x+1 est une fonction. En JS, c'est pareil : tu donnes un input, tu obtiens un output. return = "retourner" — la fonction te "rend" sa réponse. parameter = du grec para (à côté) + metron (mesure) = une valeur qui "mesure" ou configure la fonction.

Une fonction sans return retourne undefined. Si tu oublies return, ton résultat sera perdu !

🎯 Portée des variables (Scope)

scope = anglais "portée/champ d'action" (du latin scopus = but, cible). C'est la zone où une variable est "visible" et accessible. Une variable déclarée dans une fonction n'existe que dans cette fonction — elle n'est pas accessible dehors.

scope.js
let pays = "Sénégal"; // Variable GLOBALE — visible partout function maFonction() { let ville = "Dakar"; // Variable LOCALE — visible seulement ici console.log(pays); // ✅ "Sénégal" — globale accessible console.log(ville); // ✅ "Dakar" — locale accessible ici } maFonction(); console.log(pays); // ✅ "Sénégal" // console.log(ville); ❌ ERREUR — ville n'existe pas ici !

Exercice 2 — Fonctions Orange Money
Pratique : fonctions + calculs
Facile

Crée ces 3 fonctions :

  • convertirEnEuros(fcfa) — retourne le montant en € (1€ = 655 FCFA)
  • calculerFrais(montant) — retourne 1.5% du montant
  • afficherBilan(nom, montant) — affiche "Nom: X | Montant: Y FCFA | Frais: Z FCFA"
✅ SOLUTION
function convertirEnEuros(fcfa) { return (fcfa / 655).toFixed(2); // toFixed(2) = 2 décimales } function calculerFrais(montant) { return montant * 0.015; // 1.5% } function afficherBilan(nom, montant) { let frais = calculerFrais(montant); console.log(`Nom: ${nom} | Montant: ${montant} FCFA | Frais: ${frais} FCFA`); } // Tests console.log(convertirEnEuros(10000)); // "15.27" afficherBilan("Mamadou", 25000); // Nom: Mamadou | Montant: 25000 | Frais: 375

Exercice 3 — Mini Boutique
Pratique : tableaux d'objets
Moyen

Crée un tableau de 3 produits (chacun avec nom, prix, stock), puis :

  • Affiche chaque produit : "Nom: X | Prix: Y FCFA | Stock: Z"
  • Affiche le produit le plus cher
  • Calcule et affiche le prix total de tout le stock
✅ SOLUTION
const produits = [ { nom: "Téléphone", prix: 75000, stock: 5 }, { nom: "Écouteurs", prix: 15000, stock: 10 }, { nom: "Chargeur", prix: 8000, stock: 20 } ]; // Affichage produits.forEach(p => { console.log(`Nom: ${p.nom} | Prix: ${p.prix} FCFA | Stock: ${p.stock}`); }); // Produit le plus cher let plusCher = produits[0]; produits.forEach(p => { if (p.prix > plusCher.prix) plusCher = p; }); console.log(`Plus cher: ${plusCher.nom} (${plusCher.prix} FCFA)`); // Total stock let totalStock = produits.reduce((total, p) => total + (p.prix * p.stock), 0); console.log(`Valeur totale du stock: ${totalStock} FCFA`);
LEÇON 05 DOM · querySelector · getElementById
Le DOM — Sélectionner des Éléments
Apprends à cibler n'importe quel élément HTML avec JavaScript — la première étape pour rendre une page interactive.
🌳 Le DOM — L'Arbre de ta Page

Quand le navigateur charge ton HTML, il crée en mémoire une structure en arbre appelée DOM. Chaque balise HTML devient un nœud dans cet arbre.

document ← le point d'entrée TOUJOURS └── html ├── head │ ├── title │ └── link └── body ├── header │ └── h1 : "Bienvenue" ├── main │ ├── p : "Premier paragraphe" │ └── button id="monBtn" : "Cliquer" └── footer

document = l'objet global représentant toute la page HTML. C'est toujours le point de départ pour accéder au DOM. querySelector = query (requête/chercher) + Selector (sélecteur CSS) → "cherche avec un sélecteur CSS". getElementById = get (obtenir) + Element (élément) + By (par) + Id (identifiant) → "obtenir l'élément par son ID". Les noms en JS sont souvent de l'anglais condensé qui décrit exactement ce que fait la méthode.

DOM — Modifier & Créer des Éléments
Tu sais sélectionner — maintenant tu vas modifier, créer et supprimer des éléments en temps réel.
✏️ Modifier du contenu

.textContent = text + Content → modifie le texte brut. .innerHTML = inner (intérieur) + HTML → modifie le HTML à l'intérieur. .style = accède aux styles CSS inline. .classList = class + List → la liste des classes CSS de l'élément. .setAttribute = set (définir) + Attribute (attribut) → définit un attribut HTML comme src, href, alt.

🏗️ Créer des éléments — .createElement & .append

createElement = create (créer) + Element (élément HTML) → fabrique un nouvel élément HTML en mémoire. append = du latin appendere = "accrocher à, attacher à" → accroche l'élément créé à l'intérieur d'un parent. appendChild = append + Child (enfant) → ancienne méthode. prepend = pre (avant) + pend (latin : accrocher) → attache au début plutôt qu'à la fin. C'est toute une famille de méthodes qui "accrochent" des éléments à différents endroits !

LEÇON 07 events · addEventListener · projet final
Événements & Projet Final
Les événements sont le cœur de l'interactivité. Puis tu construis une vraie mini-application.
⚡ Les Événements

event = du latin eventus = "ce qui arrive, ce qui se produit". Un événement en JS = quelque chose qui se produit dans la page (clic, frappe clavier, souris qui bouge). addEventListener = add (ajouter) + Event (événement) + Listener (écouteur) → "ajouter un écouteur d'événement". Comme un vigile à la porte : il écoute et réagit quand quelque chose se passe. callback = call back = "rappeler" — une fonction qu'on donne à une autre fonction pour qu'elle la "rappelle" plus tard quand l'événement se produit.

📚 Glossaire Complet — Origines des Termes
.append()
latin : appendere = accrocher à
Accroche un élément à la FIN d'un parent dans le DOM.
.prepend()
pre (avant) + pendere (accrocher)
Accroche un élément au DÉBUT d'un parent.
.push()
anglais : pousser
Pousse/ajoute un élément à la fin d'un tableau.
.pop()
anglais : faire sauter
Retire et retourne le dernier élément d'un tableau.
.filter()
latin : filtrare = passer à travers un filtre
Retourne un nouveau tableau avec seulement les éléments qui passent le test.
.map()
cartographier = associer chaque point à un autre
Transforme chaque élément du tableau en quelque chose de nouveau.
.reduce()
latin : reducere = ramener à
Réduit un tableau à une seule valeur (somme, objet, etc.).
.find()
anglais : trouver
Trouve et retourne le PREMIER élément qui satisfait la condition.
.trim()
anglais : tailler/couper
Coupe les espaces au début et à la fin d'une chaîne.
.classList
class + List = liste de classes
La liste de toutes les classes CSS d'un élément HTML.
addEventListener
add + Event + Listener = ajouter un écouteur
Attache une fonction "écouteur" qui réagit à un événement.
callback
call back = rappeler
Fonction passée en argument à une autre, exécutée plus tard.
